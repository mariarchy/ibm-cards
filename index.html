Example from the punch-card era

Programs were written one line per card. A typical card had:

  • Columns 1–5 — Statement label (or C for comment)
  • Column 6 — Continuation (blank = new statement)
  • Columns 7–72 — Code (e.g. FORTRAN or COBOL)
  • Columns 73–80 — Sequence number so you could sort the deck if you dropped it

How it works

  • 80 columns — One character per column, left to right.
  • 12 rows — Rows are labeled 12, 11, 0, 1 … 9. Top three (12, 11, 0) are zone rows; 1–9 are digit rows.
  • Encoding — A character is a unique combination of holes in that column:
    • No holes = space.
    • Digit 0 = hole in row 0; 1–9 = hole in row 1–9.
    • A–I = zone 12 + digit 1–9 (e.g. A = 12+1, I = 12+9).
    • J–R = zone 11 + digit 1–9 (e.g. J = 11+1).
    • S–Z = zone 0 + digit 2–9 (e.g. S = 0+2, Z = 0+9).
    • Symbols use multiple punches (e.g. comma = 0-8-3).
  • History — 80-column cards (IBM, 1928) used rectangular holes for higher density. Often called “Hollerith” cards; used for data and code into the 1980s.

Encoding follows the common BCD / IBM 029–style layout. Reference: Doug Jones, Punched Card Codes.
